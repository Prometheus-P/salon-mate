## 개발 원칙

```
┌─────────┐     ┌─────────┐     ┌───────────┐
│   RED   │ ──▶ │  GREEN  │ ──▶ │ REFACTOR  │
│ (Fail)  │     │ (Pass)  │     │ (Improve) │
└─────────┘     └─────────┘     └───────────┘
     │                               │
     └───────────────────────────────┘
```

**각 기능 개발 시 반드시 준수:**
1. 테스트 먼저 작성 (RED)
2. 테스트 통과하는 최소 코드 작성 (GREEN)
3. 리팩토링 (REFACTOR)
4. 커밋
