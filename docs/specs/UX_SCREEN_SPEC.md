# UX 화면 정의서

## 1. UX 목표 & 원칙
- **정보 우선순위화**: 매장 운영자가 가장 자주 확인하는 지표(신규 리뷰, 예약 완료, 예약된 포스팅)를 상단에 배치해 눈동선 이동을 최소화.
- **즉시 행동 연결**: 모든 데이터 카드 옆에 1-2개의 CTA(예: "AI 답변 생성", "포스팅 일정 잡기")를 두어 정보 → 행동 전환 비용 최소화.
- **상태 인지성**: 데이터 동기화, 큐 작업, 승인 대기 등 비동기 상태는 토스트 + 배지 + 타임스탬프로 명시.
- **서브태스크 분할**: 리뷰 확인, 답변 작성, 게시 승인 단계를 화면 내에서 단계별로 안내해 초보자도 학습 없이 사용.

## 2. 주요 사용자 여정
1. 로그인 → 워크스페이스 선택 → 온보딩 체크리스트 완료
2. 대시보드에서 리뷰 알림 확인 → 리뷰 인박스로 이동 → AI 답변 생성/편집 → 게시 승인
3. 콘텐츠 스튜디오에서 사진 업로드 → 캡션/해시태그 생성 → 예약 발행 → 캘린더에서 일정 검토
4. 설정에서 채널 및 멤버 관리 → 알림/톤 커스터마이징 → 결제 플랜 확인

## 3. 화면 정의

### 3.1 로그인 & 온보딩 (FR-001)
- **목적**: 최소 단계로 진입, 초기 설정 완료율 90% 달성.
- **구조**
  | 섹션 | 내용 |
  |------|------|
  | Hero | 브랜드 슬로건, 데모 영상 15초 자동 루프 |
  | 로그인 폼 | 이메일/비밀번호, "계정 없음? 가입" 링크, 소셜 버튼(구글/카카오) |
  | 온보딩 Checklist Drawer | 첫 로그인 직후 슬라이드 패널, 3단계(채널 연동, AI 톤 설계, 알림 설정) 진행 상황 표시 |
- **사용성 고려**: 이메일 입력 시 즉시 검증, 소셜 로그인은 "사장님 계정으로 계속" 라벨링. 온보딩 단계는 저장 후 닫혀도 대시보드 상단 토스트로 복귀 위치 제공.
- **상태**: 로딩 스피너, 실패 시 필드 하단 인라인 에러 + 재시도 버튼, OTP 요구 시 모달.

### 3.2 메인 대시보드
- **목적**: 오늘 해야 할 일과 성과를 한 화면에서 제공.
- **카드 구성**
  | 카드 | 내용 | 행동 |
  |------|------|------|
  | 리뷰 인박스 요약 | 미응답 리뷰 수, 채널별 배지 | "인박스로 이동" CTA |
  | 콘텐츠 파이프라인 | 오늘 예약 발행 수, 드래프트 수 | "새 포스팅" |
  | 성과 지표 | 주간 리뷰 응답률, 팔로워 증가 그래프 스파크라인 | "리포트 보기" |
  | 알림 센터 | 최근 5개 시스템 알림, 읽음 토글 | "모두 읽음" |
- **상호작용**: 카드 hover 시 미니 디테일 팝오버로 다음 액션 노출, 우측 플로팅 버튼에 "빠른 작업" (리뷰 답변, 콘텐츠 생성, 알림 설정).
- **사용성 디테일**: 카드 배경 대비 4.5:1 이상, 키보드 탭 순서 카드 → CTA → 플로팅 버튼. 빈 상태는 "새 채널 연동" 일러스트 + 버튼 제공.

### 3.3 리뷰 인박스 (FR-002)
- **레이아웃**: 좌측 필터 패널, 중앙 리스트, 우측 상세 패널의 3칼럼 구조.
  - **필터**: 채널(네이버/구글), 별점 범위, 상태(미응답/초안/게시됨), 태그(불만/칭찬). 선택 시 칩 형태로 상단에 노출하고 `Cmd/Ctrl+K`로 검색 가능.
  - **리스트**: 각 리뷰 카드에 별점, 작성 채널 아이콘, 고객 이름 이니셜, Received at 표시. 미응답은 강조 테두리.
  - **상세 패널**: 상단 리뷰 내용 + 감정 분석 배지, 중단 AI 제안 답변, 하단 액션 바(톤 옵션, 예약 게시 시간, 승인 버튼).
- **AI 답변 UX**: "톤" 드롭다운(기본/친근/전문/맞춤), "답변 재생성" 아이콘 버튼, 수정 시 diff 하이라이트. 승인 시 Toast + 리스트에서 상태 뱃지 업데이트.
- **에러/비동기 처리**: 크롬 탭 전환 중 비동기 완료 시 브라우저 알림, 실패 시 상세 패널 상단 경고 리본 + "세부 로그 보기" 링크.

### 3.4 리뷰 상세 모달 (모바일 대응)
- **활성화**: 인박스 리스트에서 모바일뷰 또는 데스크탑에서도 빠른 미리보기.
- **콘텐츠 영역**: 상단 고객 메타, 중단 리뷰 본문(긴 텍스트 대비 폰트 16px), 하단 AI 답변 + 손글씨 스타일 사전보기.
- **행동**: "AI 답변 복사", "편집"(스크롤 시 고정된 CTA 바), "바로 게시". 드래그 다운 제스처로 닫기.
- **사용성**: 단일 손가락으로도 필터 접근 가능하도록 상단 시트에 필터 버튼, VoiceOver 레이블 지정.

### 3.5 콘텐츠 스튜디오 (인스타 포스팅)
- **탭 구조**: 업로드, 캡션 & 해시태그, 예약 & 미리보기.
  - **업로드 탭**: 드래그 앤 드롭, 자동 리사이즈 미리보기, EXIF에서 촬영일 자동 추출. 다중 이미지 시 순서 변경 드래그 핸들.
  - **캡션 탭**: AI가 생성한 3개의 버전 칩 형태 제시, 해시태그는 니즈(트렌드/지역/서비스)별 그룹화. 사용자는 칩을 클릭해 캡션 본문에 삽입.
  - **예약 탭**: 캘린더 + 타임피커, 우측에는 게시 미리보기(폰 프레임). 충돌 시간은 빨간 점, 추천 슬롯은 초록 배경.
- **사용성 요소**: 단계별 진행 바, 자동 저장 상태 표시(예: "초안 자동 저장 2분 전"), 이미지 업로드 실패 시 파일별 토스트 + 재시도 아이콘.

### 3.6 일정 캘린더 & 히스토리
- **보기 모드**: 주간(기본) / 월간 토글, 채널별 색상 구분.
- **카드 정보**: 포스팅 제목, 채널 아이콘, 작성자, 상태(예약/게시완료/실패). 실패 시 카드 배경 붉은 톤 + "재시도" 버튼.
- **상호작용**: 드래그 앤 드롭으로 슬롯 변경, 변경 시 확인 모달 + 자동으로 연결된 인스타/네이버 API 호출. 다중 선택으로 벌크 삭제 가능.
- **사용성**: 키보드 단축키(←/→ 주간 이동, `N` 새 포스팅). 빈 상태에는 "캘린더에 예약된 포스팅이 없습니다" 문구와 가이드 비디오 링크.

### 3.7 워크스페이스 설정
- **섹션**: 채널 연동, 멤버 & 권한, 알림, 결제.
  - **채널 연동**: 각 채널 카드에 상태 배지(정상/토큰만료/오프라인)와 "재연동" CTA, 토큰 만료 예정 시 노란 배경.
  - **멤버**: 테이블(이름, 역할, 초대 상태, 최근 활동). 역할 변경은 인라인 드롭다운, 초대 재전송 버튼.
  - **알림**: 이벤트별 토글(새 리뷰, 게시 실패, 결제), 슬랙/이메일/모바일 푸시 채널 선택.
  - **결제**: 플랜 카드, 다음 청구일, 사용량(연동 채널/멤버/콘텐츠) 프로그레스 바.
- **사용성**: 저장 시 상단 고정 토스트, 변경 사항 미저장 시 라우팅 차단 모달. 접근성 위해 토글과 설명 텍스트 연계 ID 사용.

## 4. 상태 & 에러 가이드
| 상황 | 처리 방식 |
|------|-----------|
| 채널 토큰 만료 | 대시보드 배너 + 설정>채널 카드 빨간 배경 + 이메일 알림 |
| AI 생성 지연 | 로딩 스켈레톤 + 5초 초과 시 "AI가 생각 중" 애니메이션, 15초 초과 시 대체 옵션(템플릿 불러오기) |
| 네트워크 끊김 | 전역 토스트 + 상단 연결상태 바, 폼 입력은 로컬 스토리지에 임시 저장 |
| 권한 부족 | 화면 내 섹션을 흐림 처리하고 "관리자에게 요청" 버튼 표시 |

## 5. 데이터 입력 규칙 & 컴포넌트 패턴
- 모든 주요 입력은 라벨 + 헬퍼텍스트 + 에러 텍스트 구조, 라벨과 입력을 수직 정렬해 모바일에서도 일관성 유지.
- CTA 버튼 배치: 주요 행동은 우측 하단 또는 모달 하단 고정, 보조 행동은 좌측 텍스트 버튼.
- 리스트/테이블 행 높이 64px, 모바일에서는 카드형으로 전환.
- 시간 선택은 30분 단위 스텝 기본, 드롭다운 대신 스크롤 피커 제공해 터치 정확도 향상.

## 6. 측정 지표
- 로그인→온보딩 완료 전환율 90%.
- 대시보드 진입 후 10초 내 주요 CTA 클릭률 30%.
- 리뷰 인박스에서 AI 답변 채택률 70%.
- 콘텐츠 스튜디오 초안 저장 실패율 < 1%.
